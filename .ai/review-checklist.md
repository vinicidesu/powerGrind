# Checklist de review

- [ ] Contratos não expõem dados sensíveis.
- [ ] Entrada possui validação e falhas retornam respostas previsíveis.
- [ ] Endpoints possuem autenticação/autorização coerente.
- [ ] Alterações de dados incluem migration quando aplicável.
- [ ] Segredos não são adicionados ao repositório.
- [ ] Logs permitem diagnosticar falhas sem registrar dados sensíveis.
- [ ] Testes cobrem o comportamento novo ou o risco foi documentado.
